Website Use Guide

This document describes how to use the administrative interface for the website, the primary interface for managing observations collected from phones and detection algorithm retraining.

Access

You get to the administrative interface by going to http://luminousid.com/login. Access is strictly controlled since people with access can make changes to data (but almost never delete data). New accounts created via signup on that page will need approval before they can access anything. Usernames luminousmossbosses and jane.g.smith are pre-approved at the time of this writing and are the only ones that can approve new users. For approved accounts, simply log in with a username and password.

Access level for each user can be managed at http://luminousid.com/user_list or Users in the menu on the left. There are many types of users:

  • Normal: No access, default level for new users.
  • Developer and Subadmin: Can do everything except change access level of users.
  • Admin: Can do everything include change access level of users.

The far right column on the Users page has select boxes that allow admins to change the type of any user.

Note that it is very dangerous to change someone to admin since they could theoretically revoke everyone else’s access, which would require directly repairing the database to fix. We recommend only having one admin which is the owner of the program.

Observations

Observations are listed at http://luminousid.com/observation_list or Observations in the menu on the left. The first 661 observations were imported by us in order to train our first detection algorithm. All others come from phones. The User ID column is a unique name given to each installation of the mobile app. The far right column allows admins, subadmins, and developers to verify an observation as silene or not. The tabs at the top allow filtering by verification. In particular, the unconfirmed tab is useful for viewing new observations just uploaded by phones. Clicking the filename will open up a more detailed view of the observation, with location, accuracy, and date information. Additionally, at the bottom is a select box called “Use Image For” allowing you to include the image from that observation in the training or testing set.

CSV

Observations can be downloaded at http://luminousid.com/csv or CSV Download in the menu to the left. A preview of the data will be displayed on that page. Pressing “Save CSV” will start a download of the actual csv file. Names of columns in the actual csv file are more informative than the preview. Only observations from phones are included in the CSV file.

Blacklisting

Hooligans sending inappropriate observations? You can blacklist them! Go to http://luminousid.com/device_list or Devices on the menu to the left, the last column can be used to add and remove blacklistings for devices. A blacklisted device can still pretend to send observations, and to them nothing has changed, but in the future the website will toss their observations without recording them.

Retraining

The bag of words algorithm can be retrained by going to http://luminousid.com/xml or Training->XML Generator in the menu to the left. Two buttons exist:

  • Retrain Bag of Words: Starts the retraining scripts. They may take a few hours to run and may make the website a little laggy. Do not click again until it has completed. It will use all the images in the database that are marked as training and have been verified.
  • Download Current Bag of Words: Downloads a copy of the bag of words xml files. See Android and iOS documentation to see how to use them. Do not downloaded while the retraining is running.

You will need to refresh in order to see if the retraining is still running or not.
